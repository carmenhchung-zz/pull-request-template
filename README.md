# Introduction

This repo provides a [pull request template](https://github.com/carmenhchung/pull-request-template/blob/master/pull_request_template.md) that can be automatically applied to every pull request.

Pull request templates are incredibly useful: they allow developers to write consistent, clear pull request descriptions that are helpful to those (like new hires or code reviewers) who are required to review hundreds of lines of code without any context (ğŸ˜­). More importantly, a PR template acts as a safeguard to ensure that team members don't accidentally forget steps before, during, and after merging and deployment.

If you've ever been lost for words when writing a pull request description (ğŸ¤”), or forgotten to migrate that data after you deployed (ğŸ¤¦ğŸ»â€â™€ï¸!), or thought you'd write a spec later for your code but failed to do so (ğŸ˜ ), then this is a great starting point.
